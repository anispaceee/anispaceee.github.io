# ANISpace 功能优化设计文档

## 第一批：用户头像点击 + 个人主页重构

### 功能 1：用户头像点击进入个人主页

**需求**：所有显示用户头像的位置（世界频道、论坛帖子、回复、通知等）均可点击进入该用户个人主页。

**隐私控制**：
- 在个人设置中新增"允许其他人查看主页"开关
- 开关开启：其他用户点击头像可进入个人主页，查看标记记录
- 开关关闭：点击头像时弹出提示"该用户已设置隐私保护"
- 自己查看自己的主页不受限制

**实现方案**：

1. **创建可点击头像组件 `UserAvatar`**（替代所有场景中的 `<img>` 头像）
   - 位置：`src/components/Common/CommonComponents.jsx`
   - Props：`userId`, `src`, `alt`, `size`, `onClick`
   - 行为：点击时检查目标用户的 `allow_profile_view` 设置
     - 允许：导航到 `/profile/{userId}`
     - 不允许：显示 toast 提示
   - 自己的头像始终可点击

2. **后端支持**：
   - `users` 表新增 `allow_profile_view` 字段（默认 true）
   - `users` 表新增 `allow_comments_public` 字段（默认 true）
   - Worker 新增 `GET /api/users/:id/profile` 端点，返回公开信息（受隐私设置控制）
   - Worker 新增 `PUT /api/users/:id/settings` 端点，更新隐私设置

3. **替换所有头像渲染位置**：
   - `WorldChannel.jsx` - 消息头像
   - `Forum.jsx` / `PostDetail.jsx` - 帖子/回复作者头像
   - `HomePage.jsx` - 热门帖子/世界频道头像
   - `Notifications.jsx` - 通知头像
   - `FriendSpace.jsx` - 好友头像
   - `Mailbox.jsx` - 邮件头像

### 功能 2：个人主页 UI 重构

**参考**：bgm.tv 用户页面风格

**布局**：左侧边栏（220px）+ 右侧主内容

#### 左侧边栏内容（从上到下）：

1. **头像 + 编辑/设置入口**
   - 头像居中显示
   - 头像右上角两个小圆形按钮：✏️编辑资料 / ⚙️设置
   - 仅在自己的主页显示
   - 昵称、签名

2. **数据统计**（竖排）
   - 📺 动画 X | 🎮 游戏 X | 📖 小说 X | ⭐ 均分 X.X

3. **标记进度**
   - 堆叠进度条（想看/在看/看过/搁置/抛弃）
   - 下方图例 + 数字

4. **登录活跃度**（年统计热力图）
   - 类似 GitHub 贡献图
   - 仅显示年度统计
   - 数据来源：`collections` 表的 `created_at`/`updated_at`

#### 右侧主内容：

1. **想看** - 单行横滑封面 + "更多→"
2. **在看** - 单行横滑封面 + "更多→"
3. **看过** - 单行横滑封面 + "更多→" + 下方最近评论
4. **搁置** - 单行横滑封面 + "更多→"（折叠，默认不展开）
5. **抛弃** - 单行横滑封面 + "更多→"（折叠，默认不展开）

**封面样式**：与搜索结果 SubjectCard 一致的 3:4 比例圆角矩形

**"更多"功能**：点击后展开该分类的完整网格视图

**最近评论**：
- 抓取用户在条目详情页提交的评分+评论（`ratings` 表数据）
- 不是论坛发言
- 可在设置中控制是否对外公开（`allow_comments_public`）

#### 隐私设置（在设置弹窗中）：
- "允许其他人查看主页"开关
- "公开我的评论"开关

#### 响应式：
- < 900px：侧边栏收起为顶部横排
- < 600px：进一步简化

### 数据库变更

```sql
-- users 表新增字段
ALTER TABLE users ADD COLUMN allow_profile_view INTEGER DEFAULT 1;
ALTER TABLE users ADD COLUMN allow_comments_public INTEGER DEFAULT 1;
```

### Worker API 变更

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/users/:id/profile` | GET | 获取用户公开信息（受隐私设置控制） |
| `/api/users/:id/settings` | PUT | 更新用户隐私设置 |
| `/api/users/:id/comments` | GET | 获取用户对条目的评论（受隐私设置控制） |

## 第二批：世界频道重构 + 搜索优化 + 百科精简

### 功能 3：世界频道重构为长帖子

**定位**：世界频道 = 一个可持续回复互动的长帖子

**实现方案**：
- 复用论坛 PostDetail 的回复系统
- 世界频道消息 = 帖子回复
- 保留实时聊天 UI 风格
- 新增"加载更多"分页
- 后端 API：`GET /api/world-messages?limit=20&offset=0`

### 功能 4：首页搜索优化

- 移除首页独立的搜索栏组件
- 强化右上角全局搜索栏（GlobalSearch.jsx）
- 搜索结果添加"更多"按钮，点击跳转百科搜索页

### 功能 5：百科页面精简

- 移除"排行榜"模块（RANK_TABS, fetchRankings 等）
- 移除"随机推荐"模块（randomItem, fetchRandomItem 等）
- 保留搜索功能

### 功能 6：首页 7:3 分栏布局

**保留模块**：随机推荐、业界资讯、每日放送、热门帖子、世界频道
