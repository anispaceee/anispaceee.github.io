## 1. 竞品概览

### 1.1 AniBT (anibt.net)

| 维度 | 分析 |
|------|------|
| **定位** | 番组放送时刻表 — 聚焦当季番剧时间线 |
| **核心功能** | 按星期展示当季番剧播出时间、制作公司、类型标签（漫画改/轻小说改/原创等） |
| **数据源** | 基于 Bangumi/AniList 数据，自动同步当季番组 |
| **交互** | 简洁的时间线布局，按日期/时段排列，点击进入条目详情页 |
| **社区** | 无社区功能，纯工具型 |
| **技术** | 传统 SPA，服务端渲染 |
| **与 ANISpace 重叠** | 当季番剧时间线（ANISpace 的 Worldline/毒电波已有类似功能） |
| **差异化机会** | AniBT 纯工具无社区；ANISpace 可结合时间线+讨论+评分形成闭环 |

### 1.2 同萌 / ai2.moe

| 维度 | 分析 |
|------|------|
| **定位** | ACG 同萌社 — 传统 BBS 论坛型二次元社区 |
| **核心功能** | 分区论坛（新人考核、闲聊水区、技术交流、资源分享、动漫讨论、游戏区、Galgame 区等） |
| **社区机制** | 新人考核制（需在"酒馆"获得门票）、虚拟货币"萌汁"、等级系统 |
| **技术** | IPS (Invision Community) 论坛系统，传统 PHP |
| **内容量** | 闲聊区 287.9K 帖子，新人考核区 158.5K 帖子，活跃度较高 |
| **与 ANISpace 重叠** | 论坛（放課後）、Galgame 讨论、资源分享 |
| **差异化机会** | 同萌是传统论坛，UI 老旧；ANISpace 桌面窗口化 UI 更现代。同萌的"考核制"门槛高，ANISpace 更开放。同萌无追番/评分系统 |

### 1.3 紫缘社 / galzy.moe

| 维度 | 分析 |
|------|------|
| **定位** | Galgame 专用数据库 — 标签化检索与评分 |
| **核心功能** | 收录 1968 条 Galgame 条目、标签系统（超细粒度：如"姐/妹控主人公""雌小鬼""病娇"等 30+ 热门标签）、每周热门排行 |
| **数据模型** | 每个条目有详细标签、封面、评分，类似 VNDB 的中文版 |
| **社区** | 极弱，主要是浏览和检索，无论坛/评论 |
| **技术** | 静态站+数据库查询 |
| **与 ANISpace 重叠** | Galgame 条目数据（ANISpace 的禁書目錄/Wiki 有 Bangumi 条目，但 Galgame 标签粒度远不如紫缘社） |
| **差异化机会** | 紫缘社标签体系非常专业（比 Bangumi 细得多），ANISpace 可考虑对接其标签数据或 API。紫缘社无社区/追番/评分闭环 |

### 1.4 ACGDB / acgdb.de

| 维度 | 分析 |
|------|------|
| **定位** | ACG 资源目录 — OpenList 驱动的网盘索引 |
| **核心功能** | 网盘资源目录列表，类似 ACG 资源导航 |
| **技术** | OpenList（开源网盘列表程序），部署在德国 .de 域名 |
| **运营** | 已存活 1076+ 天，有 Telegram 群 |
| **社区** | 无，纯资源索引 |
| **与 ANISpace 重叠** | 资源聚合（ANISpace 的视频聚合/TouchGal 有类似定位） |
| **差异化机会** | ACGDB 是纯链接索引，无播放/预览能力；ANISpace 可内嵌播放+评分形成体验闭环 |

## 3. ANISpace 核心差异化优势

1. **全功能闭环**：唯一同时具备追番+社区+资源+音乐+AI+创作的平台
2. **桌面化 UI**：窗口+Dock 交互范式，竞品均为传统网页布局
3. **开放 APP 平台**（规划中）：竞品均无可扩展性，ANISpace 将是 ACG 领域首个支持用户自开发 APP 的平台
4. **武藏创作平台**：竞品均无用户原创内容发布能力

## 5. 对 App Store 规划的启示

基于竞品分析，App Store 的定位应强调：

1. **填补竞品空白**：所有竞品均为封闭系统，App Store 的开放性是独有卖点
2. **生态互补**：可开发"AniBT 时间线 APP""紫缘社标签检索 APP"等桥接竞品能力的第三方 APP
3. **社区驱动**：同萌的虚拟货币体系可借鉴为 APP 开发者激励（下载量→贡献值→特权）
4. **资源聚合**：ACGDB 的 OpenList 模式可通过 APP 形式接入 ANISpace，避免主站承担版权风险
