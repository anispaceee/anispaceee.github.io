# 游客功能开放设计方案

> 日期: 2026-06-13
> 状态: 待实施

## 1. 背景与目标

当前 ANISpace 大部分社交功能需要登录才能使用，游客只能浏览首页和条目信息。本方案旨在开放更多只读浏览功能给游客，同时引入本地存储机制让游客可以记录进度/收藏/点赞，并在登录后提供数据合并能力。

**目标：** 降低使用门槛，让游客先体验核心功能价值，再引导登录。

## 2. 只读浏览开放

| 功能 | 游客权限 | 登录用户权限 |
|------|---------|------------|
| 放課後帖子 | 浏览帖子列表和详情 | + 发帖/评论/点赞 |
| Tea Time! | 浏览社团列表和聊天记录 | + 发言/加入/创建 |
| LeMU 动态 | 浏览好友动态流 | + 发动态/点赞 |
| 世界线消息 | 浏览消息流 | + 发言 |
| 条目详情页 | 浏览信息+评论（只读） | + 评分/收藏/评论/点赞/收录 |

### 实现方式

- 移除各组件中阻止游客浏览的 `isAuthenticated` 检查
- 保留写操作（发帖/评论/发言等）的 `isAuthenticated` 检查，改为显示"登录后操作"提示
- 游客看到输入框时显示占位提示"登录后发言"，点击触发 openAuth()

## 3. 游客本地存储

### 3.1 存储内容

| 数据类型 | localStorage Key | 结构 |
|---------|-----------------|------|
| 条目进度标记 | `acg_guest_marks` | `{ [subjectId]: 'wish' | 'done' | 'hold' | 'drop' }` |
| 条目收藏 | `acg_guest_favorites` | `{ [subjectId]: true }` |
| 评论点赞 | `acg_guest_likes` | `{ [commentId]: true }` |
| 条目评分 | `acg_guest_ratings` | `{ [subjectId]: number }` |

### 3.2 存储策略

- 所有数据存储在 localStorage，key 统一使用 `acg_guest_` 前缀
- 游客点赞不计入服务端统计，仅本地记录
- 游客评分仅本地显示，不影响服务端评分均值
- 条目详情页读取时：优先显示服务端数据（已登录），否则降级到本地数据（游客）

### 3.3 UI 表现

- 游客点击"想看/看过/搁置/丢弃"按钮时，数据存入 localStorage，按钮状态正常切换
- 游客点击收藏按钮时，数据存入 localStorage，按钮状态正常切换
- 游客点击点赞按钮时，数据存入 localStorage，点赞数+1（本地显示）
- 游客点击评分时，数据存入 localStorage，评分正常显示
- 所有写操作不再弹 openAuth()，而是静默存入本地

## 4. 登录合并

### 4.1 合并触发

- 用户从游客状态登录成功后，检测 localStorage 中是否存在 `acg_guest_*` 数据
- 如果存在，弹出确认弹窗：

```
┌──────────────────────────────────────┐
│  📋 检测到本地浏览记录               │
│                                      │
│  你在游客状态下记录了：              │
│  • 3 个条目进度标记                  │
│  • 2 个收藏                          │
│  • 5 个点赞                          │
│  • 1 个评分                          │
│                                      │
│  是否将这些记录合并到你的账号？      │
│                                      │
│  [合并到账号]  [仅保留服务端数据]    │
└──────────────────────────────────────┘
```

### 4.2 合并策略

- **进度标记**：如果服务端已有标记，保留服务端数据；如果服务端无标记，写入本地标记
- **收藏**：取并集（本地+服务端都保留）
- **点赞**：将本地点赞逐项同步到服务端（调用点赞 API）
- **评分**：如果服务端已有评分，保留服务端数据；如果服务端无评分，写入本地评分

### 4.3 合并后清理

- 合并成功后，清除 `acg_guest_*` localStorage 数据
- 如果用户选择"仅保留服务端数据"，也清除本地数据

## 5. 个人主页游客提示

- 游客访问个人主页（/profile 或点击头像）时显示提示条：

```
⚠️ 当前为游客状态，删除浏览器缓存后记录消失
[登录保存数据]
```

- 提示条固定在页面顶部，可关闭（关闭后本次会话不再显示）
- "登录保存数据"按钮触发 openAuth()

## 7. 优先级与实施顺序

1. **P0 - 只读浏览开放**：移除各组件的浏览限制，让游客能看到内容
2. **P1 - 游客本地存储**：实现 GuestStorageService，条目详情页支持游客操作
3. **P2 - 登录合并**：实现合并弹窗和合并逻辑
4. **P3 - 个人主页提示**：添加游客状态提示条
